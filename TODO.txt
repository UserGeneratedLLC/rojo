[Always mode]
MEDIUM - Issue 5: Ongoing "Always" mode patches don't support per-item selections yet
The existing code at ServeSession.lua line 311-314 explicitly says:
> "Table-type responses from the selection-based UI are not yet handled here. Ongoing WebSocket patches with per-item selections would need the same logic."
So in "Always" mode, ongoing patches currently only support "Accept"/"Abort"/"Skip" -- not per-item push/pull/ignore. The git defaults would be set in the PatchTree but the user can't act on them for ongoing patches. This is a pre-existing limitation, not introduced by us, but worth noting.
